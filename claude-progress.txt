### Completed Features:
- Feature #233: Timestamps display in Chile timezone ✅
- Feature #234: Date picker defaults to today in local timezone ✅
- Feature #235: Audit trail timestamps in chronological order ✅

### Changes Made:
- frontend/src/lib/formatters.js: Major timezone overhaul
  - Added CHILE_TZ constant ('America/Santiago')
  - Added parseDate() helper to correctly handle:
    - Date-only strings (YYYY-MM-DD) → parsed as noon UTC to avoid day shift
    - SQLite datetime strings (YYYY-MM-DD HH:MM:SS) → treated as UTC
    - ISO 8601 strings → parsed as-is
  - formatDate(), formatDateTime(), formatRelativeDate() all use timeZone: CHILE_TZ
  - Added getTodayInChile() for date picker defaults
- frontend/src/pages/Ingresos.jsx: Replaced new Date().toISOString().split('T')[0] with getTodayInChile()
- frontend/src/pages/Solicitudes.jsx: Added formatDateTime import, used it for Excel export dates

### Key Technical Detail:
- SQLite CURRENT_TIMESTAMP stores UTC, but browsers parse space-separated datetime strings
  as local time. The parseDate() function converts "YYYY-MM-DD HH:MM:SS" to "YYYY-MM-DDTHH:MM:SSZ"
  to ensure correct UTC interpretation before Chile timezone conversion.
- Date-only strings "YYYY-MM-DD" are parsed as noon UTC to prevent timezone conversion
  from shifting the date backwards (midnight UTC → previous day in UTC-3).
